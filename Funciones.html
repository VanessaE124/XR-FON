Bienvenido a la documentación de XR-FON

  • Somos una librería de realidad extendida para dispositivos con sistema operativo Android. Permite el seguimiento de un objeto físico fijo (mesa), de una dimensión variable de 20 cm^2 a 2 m^2, y dos objetos físicos rígidos que pueden ser ubicados sobre la mesa; y que pueden ser manipulados en el entorno virtual por el usuario.

    • Funciones

  • Funcionar en dispositivos móviles tipo Android
  • Está desarrollado para el motor de videojuegos Unity
  • Visualización de un entorno virtual
  • Interacción de objetos del mundo real en el entorno virtual

    • Los prerrequisitos son fundamentales

  • Aquí encontrarás tanto las funciones de nuestra librería como una introducción para modificar las mismas según tu necesidad. Lo haremos describiendo los scripts y las posibles acciones con targets y modelos 3D.

    Botón virtual

    Se crean dos UnityEvent, OnButtonPressed y OnButtonReleased, los cuales serán los eventos que almacenarán las acciones al momento de oprimir el botón; además del Gameobject, Habitación, que sirve para enlazar el modelo del entorno al virtual button. Por consiguiente, se crea la función private void OnEnable que registra cuando el botón es presionado y cuando no, a su vez, dentro de esta, se desactiva el GameObject para que solo se muestre el entorno virtual en caso de que llegue a ser oprimido el botón.

    • Target botón

    ButtonPressed

    En la función ButtonPressed, se llama al evento OnButtonPressed, que, mediante un GameObject.SetActive, activa la habitación; por último, se agrega un Debug.Log para que se envíe un mensaje en caso de que el botón sea presionado.

    ButtonReleased

    Por su parte, en la función ButtonReleased, se activa el UnityEvent OnButtonReleased, el cual tiene los mismos parámetros que el OnButtonPressed; este se encuentra disponible por si el usuario desea agregar una sección o función al evento. Para finalizar se crea la función OnDestroy, que sirve para remover las instancias del ButtonPressed en el caso de que se active el ButtonReleased y viceversa.

    1.) Base de datos

    Los targets se instalan en Unity como un package que contiene una base de datos de targets, esta base de datos se crea desde Vuforia. Para cambiar estos targets que vienen por defecto en la librería se debe cambiar la base de datos, por lo que se necesitarán realizar los siguientes pasos:

    1. Ingresar al sitio web TargetManager de Vuforia: Enlace
    2. Inicia sesión con el correo y la contraseña registrados al instalar y obtener la licencia en Vuforia.
    3. Ingresar a la pestaña Target Manager.

    2.) Agregar base de datos

    Dar clic en el botón Agregar Base de datos, ingresar el nombre y dar clic en crear

    3.) Agregar targets a la base de datos

    1. Dar clic en el botón Agregar target
    2. Completar las especificaciones del target para luego dar clic en Añadir.
    3. En caso de que sea un target diferente a un plano, seleccionar cada cara del target y añadirle la imagen deseada.

    4.) Descargar base de datos

    1. Una vez creados los targets necesarios, descargar la base de datos.
    2. Seleccionar la opción Unity Editor y descargar.

    5.) Importar la base de datos en Unity

    1. Importar el package en el proyecto en Unity.
    2. En import packager y dar en Importar

    6.) Ajustar el target y el modelo

    1. Seleccionar el elemento del target y dirigirse al inspector.
    2. En Database seleccionar la base de datos correspondiente.
    3. En Image Target seleccionar el target correspondiente.
    4. Ajustar el modelo al target como desee.
